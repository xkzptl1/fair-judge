# Layout Specification

## Overall Principle  
  
The UI must prioritize understanding structure over consuming content.  
  
---  
  
## Home Screen  
  
Purpose:  
- Show topics, not articles  
  
Each topic card must include:  
  
- topic title  
- article count  
- source count  
- last updated  
- stance distribution  
- visual bar  
- fact-check badge  
  
Users should immediately see:  
- whether the topic is controversial  
- whether reporting is mostly neutral  
  
---  
  
## Topic Detail Screen  
  
### Section Order  
  
1. Topic Header  
2. Stance Distribution  
3. Main Issues Summary  
4. Fact Check Section  
5. Stance Tabs  
6. Article List  
  
---  
  
## First Viewport Rule  
  
The first screen MUST show:  
  
- topic context  
- stance distribution  
- main issues  
  
Users should understand the topic before scrolling.  
  
---  
  
## Stance Distribution  
  
Display:  
  
- support count  
- challenge count  
- report count  
- unclear count  
  
Include a visual bar.  
  
---  
  
## Main Issues Section  
  
Short bullet points summarizing key arguments.  
  
Example:  
  
- safety concerns  
- economic impact  
- regulatory differences  
  
---  
  
## Stance Tabs  
  
Tabs:  
  
- Support  
- Challenge  
- Reporting  
- Unclear  
  
---  
  
## Article Cards  
  
Each card must include:  
  
- source  
- headline  
- short summary  
- classification reason  
- confidence (optional)  
- link to original  
  
---  
  
## Comparison Feature  
  
Users must be able to compare perspectives.  
  
Provide:  
  
- compare button  
- structured difference view  
  
---  
  
## Fact Check Section  
  
Separate from article grouping.  
  
Never mix with stance categories.}
